# Agent Process Backlog

当前 0.1 alpha 先保留最小可用 Agent 角色，不扩大实例表。

后续如果系统从原型进入长期运行，应补充这些流程能力：

1. 数据接入与回填：拉取 BC、Binance、日历等历史和实时数据。
2. 数据质量与时间守门：检查 ET、UTC+8、DST、缺失数据、复权和 no-lookahead。
3. 证据链管理：保存原文、截图 OCR、行情片段、回测结果并生成 evidence_id。
4. 纵坐标计算：根据纵坐标定义生成具体价格、阈值、A/B/C、前高前低等实例。
5. 事件注入：把 FOMC、假期、宏观事件、BTC 急波动等转为横坐标节点。
6. 实时 Path Runtime：推进 path_instance 的节点状态、失效状态和结果观测。
7. 实验设计：在回测前定义样本范围、对照组、外样本切分和评价指标。
8. 告警与日报：压缩实时状态、触发原因、数据延迟和运行耗时。

这些先作为后续流程设计备忘，不写入当前 Agent 实例表。
